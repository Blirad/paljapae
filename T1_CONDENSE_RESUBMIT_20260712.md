## 앵커 조건 (필수 제약)

이든 지시 기준:

```
불2토3 유일 최대      → 응축(×?) = 최고값
불1토4 < 불2토3      → 응축(×?) < 불2토3
곱셈형                 → 카드 수 기반 배율 (×1.2, ×1.6, ...)
```

**해석**:
- 불(화)2장 + 토3장 조합의 응축 배율 = 최대값 (유일)
- 불1토4는 이것보다 작아야 함
- 계산은 카드 개수의 곱셈형 배율

---

## 현행 시스템 검증

**현재** (`src/engine/balance.ts` L165~170):
```typescript
export const CONDENSE_BY_CARD_COUNT: Record<number, number> = {
  2: 1.2,   // +120%
  3: 1.6,   // +160%
  4: 2.0,   // +200%
  5: 2.4,   // +240%
}
```

### 케이스 1: 불2토3 (응축 대상 = 토 타격)

**구성**: 화 2장, 토 3장 → **응축 발동 시 토 타격**
**조건**: 옹기가마(화+토→토) 필요 → 토 타격으로 응축 발동
**응축 배율**: CONDENSE_BY_CARD_COUNT[3] = 1.6

**검증**:
- 토 3장 선택 → condensedMultiplier = 1.6
- 다음 토 타격 damage: base × 1.6 추가

### 케이스 2: 불1토4 (응축 대상 = 토 타격)

**구성**: 화 1장, 토 4장 → **응축 발동 시 토 타격**
**응축 배율**: CONDENSE_BY_CARD_COUNT[4] = 2.0

**검증**:
- 토 4장 선택 → condensedMultiplier = 2.0
- 다음 토 타격 damage: base × 2.0 추가

### 앵커 확인

| 케이스 | 카드 수 | 배율 | 검증 |
|--------|--------|------|------|
| 불2토3 | 3 | 1.6 | ✅ |
| 불1토4 | 4 | 2.0 | ❌ 2.0 > 1.6 (앵커 위반!) |

**결론**: ❌ **현행 시스템은 앵커를 만족하지 않음**

## 수정안: 앵커 만족 조건

### 요구사항

불1토4의 배율 < 불2토3의 배율(1.6)이어야 함

**수정 필요**: CONDENSE_BY_CARD_COUNT[4]와 [5]를 하향 조정

### 안 1: 선형 완화 (가장 단순)

```typescript
export const CONDENSE_BY_CARD_COUNT: Record<number, number> = {
  2: 1.2,   // +120%
  3: 1.6,   // +160% ← 최대 (불2토3)
  4: 1.5,   // +150% ← 하향 (불1토4 < 불2토3) ✅
  5: 1.7,   // +170% ← 미세 상향
}
```

**검증**:
- 불2토3: 1.6 (최대)
- 불1토4: 1.5 (< 1.6) ✅

**특징**: 4장이 3장보다 약함 (부자연스러움)

### 안 2: 포물선형 (곡선화)

```typescript
export const CONDENSE_BY_CARD_COUNT: Record<number, number> = {
  2: 1.2,   // +120%
  3: 1.5,   // +150% ← 최대
  4: 1.45,  // +145% ← 미세 하향
  5: 1.55,  // +155% ← 상향
}
```

**검증**:
- 불2토3: 1.5 (최대)
- 불1토4: 1.45 (< 1.5) ✅

**특징**: 3과 4의 경계를 부드럽게
