## 부기운극 코드 분석

### 코드 (paljajeonEngine.ts L273-283)

```typescript
// R8 복원: 부 기운 극 보너스 ×1.25
// (주 기운 극 미적용 시, 카드가 부 기운을 극하면 적용)

const mainGeukApplied = floorEnemyEl ? GEUK_MAP[repEl] === floorEnemyEl : false
if (!mainGeukApplied && subEl) {  // ← 주극 미적용 시에만 실행
  const hasSubGeuk = playedCards.some(c => GEUK_MAP[c.element] === subEl)
  if (hasSubGeuk) {
    damage = Math.round(damage * SUB_GEUK_BONUS)  // ×1.25
  }
}
```

### 부기운극 특성: **대체(Replacement)** ✅

**근거**:
```
if (!mainGeukApplied && subEl) {...}
```

- 조건: `!mainGeukApplied` = 주 기운 극이 **미적용**되었을 때만
- 즉, 주극과 부극은 동시에 적용되지 않음
- 부극은 주극을 **대체**하는 것이지, **중첩**되지 않음

## 실계산 예시: 수→목 + 부기운 화

### 시나리오 설정

**아군**:
- 수(水) 카드 2장 (각 값 4)
- 기본 콤보 = 낱장×2 (모으기 판정 실패, 낱장으로 간주)
- 우 2장 따로 공격하는 경우

**적군** (3층 정예 고신):
- 주 원소: 목(木)
- 부 원소: 화(火)
- 상성 판정:
  - 주극: 수가 목을 극하는가? → GEUK_MAP['su'] = 'hwa' ≠ 'mok' → **미적용**
  - 역극: 목이 수를 극하는가? → GEUK_MAP['mok'] = 'to' ≠ 'su' → 미적용
  - 역생: 목이 수를 생하는가? → SANG_MAP['mok'] = 'hwa' ≠ 'su' → 미적용
  - **동기**: 매칭 없음 → ×1.0

**1번 공격 (수 카드 1장)**:
```
rawBase = 4
× 낱장 배율 = 4 (낱장 특별 배율 없음, 기본만)
× 상성 동기 ×1.0 = 4
× 부극 체크:
  - mainGeukApplied = false (수가 목을 극하지 않음)
  - hasSubGeuk = (수가 화를 극하는가?) = true (GEUK_MAP['su'] = 'hwa')
  - 부극 ×1.25 적용 ✅
= 4 × 1.25 = 5 → 최종 5
```

**2번 공격 (수 카드 1장, 동일)**:
```
rawBase = 4
× 상성 동기 ×1.0 = 4
× 부극 ×1.25 = 5 → 최종 5
```

### 배율 분석

| 단계 | 배율 | 누적 |
|------|------|------|
| 기본 damage | 4 | 4 |
| 상성 (동기) | ×1.0 | 4 |
| 부극 (대체) | ×1.25 | **5** |
| 용신 | × (미적용) | 5 |

**결론**: 부기운극은 주극의 **대체**이며, 수→목 공격 시 화 부극이 있으면 ×1.25 적용.
