## 근거

### 코드 정의 (balance.ts)

**GEUK_MAP** (극 관계):
```typescript
export const GEUK_MAP = {
  mok: 'to',      // 목 극 토
  hwa: 'geum',    // 화 극 금 ← 화가 목을 극하지 않음
  to: 'su',       // 토 극 수
  geum: 'mok',    // 금 극 목
  su: 'hwa',      // 수 극 화
}
```

**SANG_MAP** (생 관계):
```typescript
export const SANG_MAP = {
  mok: 'hwa',     // 목 생 화
  hwa: 'to',      // 화 생 토 ← 화가 목을 생하지 않음
  to: 'geum',     // 토 생 금
  geum: 'su',     // 금 생 수
  su: 'mok',      // 수 생 목 ← 수가 목을 생함 (역생)
}
```

### 화→목 분류 체크

| 관계 | 코드 | 결과 |
|------|------|------|
| 화가 목을 극하는가 | GEUK_MAP['hwa'] === 'mok' | ❌ (hwa극geum) |
| 화가 목을 생하는가 | SANG_MAP['hwa'] === 'mok' | ❌ (hwa생to) |
| 목이 화를 극하는가 | GEUK_MAP['mok'] === 'hwa' | ❌ (mok극to) |
| 목이 화를 생하는가 | SANG_MAP['mok'] === 'hwa' | ✅ **역생** |

### 결론

화→목은:
- **극 아님** (화는 금을 극함)
- **생 아님** (화는 토를 생함)
- **역생** (목이 화를 생함 = 화 입장에서는 역생)
- **배율**: ×1.0 (무효과)

## 수정된 5×5 상성 매트릭스

**표 읽기**: 행(세로)=내 타격 원소 / 열(가로)=적 주 원소

|       | 목   | 화   | 토   | 금   | 수   |
|-------|------|------|------|------|------|
| **목** | ×1.0 | ×1.0 | ×1.7 | ×0.75| ×1.0 |
| **화** | ×1.0 | ×1.0 | ×0.5 | ×1.7 | ×0.75|
| **토** | ×0.75| ×1.0 | ×1.0 | ×0.5 | ×1.7 |
| **금** | ×1.7 | ×0.75| ×1.0 | ×1.0 | ×0.5 |
| **수** | ×0.5 | ×1.7 | ×0.75| ×1.0 | ×1.0 |

**변경 항목**: 화 행(세로) 목 열(가로) = ×0.5 → **×1.0**

## 오류 원인

상생 관계 SANG_MAP에서 "key가 value를 생함"의 정의를 잘못 적용함.

**올바른 해석**:
- `SANG_MAP['hwa'] = 'to'` = 화가 토를 생함
- 화→목의 상생 관계를 찾으려면 `SANG_MAP[?] = 'mok'`을 확인해야 함
- `SANG_MAP['su'] = 'mok'` ← **수가 목을 생함** = 화는 역생
