## 수정 2 — 응축 clamp 완화 (기준값·하한)

### 현행 코드 (fullCapBot.ts 또는 balance.ts)

```typescript
const effectiveness = MATRIX% * Math.max(0.4, Math.min(1.0, condenseCounts / 15));
```

### 수정 규격

```typescript
const effectiveness = MATRIX% * Math.max(0.6, Math.min(1.0, condenseCounts / 10));
```

변경:
- 기준값: 15 → **10** (투입값합 기준)
- 하한: 0.4 → **0.6** (저치 착취 차단 강화)

### 논리

토단일은 구조적으로 응축 투입값(화N + 토M)이 낮다.
- 기준값 15 → 10으로 낮추면: 낮은 투입도 효율성이 올라감
- 하한 0.4 → 0.6으로 높이면: 최악의 경우도 60% 이상 실효

결과: 저투입(토단일 5~8장)도 주력기를 죽이지 않는 선에서 작동

## 수식 버그 검증 (측정기 무죄 우선)

```
baseValue = 현재 핸드 최고 공격 조합 점수
```

이 `baseValue`에 다음이 포함되었는가?
- (a) 상성 배율 (상극 ×1.5 등)
- (b) 조건부 배율 (카드 종류, 상황별)

### 검증 방법

1. 코드에서 즉발 계산 함수 위치 특정

3. 포함되면: (a)버그 판정 (잔불 기대값도 상성 적용해야 함)
4. 미포함되면: (b)설계 문제 판정 (즉발도 상성 미적용이 설계)
